# log_manage.html 개편 계획 (보류 — worklog 축적 후 구현)

> **상태:** 계획 확정, 구현 보류.
> worklog.md가 2~3주 쌓인 뒤 이 문서대로 구현 예정. 그 전엔 코드 수정 없음.
> 작성일: 2026-07-03

## 배경 (왜)
- `admin/dashboard/log_manage.html`(작업 로그 관리)이 방치된 진짜 이유 = "전부 수동 입력"이라 귀찮아서.
- 작업 로그의 정본(SSOT)은 repo의 `worklog.md`로 정함. Claude Code가 작업 종료 시 자동으로 한 줄 남기게 이미 세팅됨.
- 목표: log_manage를 "손으로 채우는 입력 폼"이 아니라 "worklog가 자동으로 흘러들어와 보이는 화면(창문)"으로 전환.

## 방향 (SSOT → 창문, 단방향)
- `worklog.md` = 정본(SSOT). `log_manage` = 그걸 받아 보여주는 뷰.
- 흐름은 한 방향만: [작업 끝남] → [worklog.md 한 줄] → [log_manage 자동 표시].
- **양방향 동기화 금지.** log_manage에서 수정한 걸 worklog로 되돌리는 구조는 만들지 않는다(싱크 깨짐 방지).
- 기존 걸 고쳐서 재활용. 새 페이지 만들지 않는다.

## A. 현재 구조 요약
- **데이터 출처:** GAS WebApp(`CHEESE_ADMIN_API_BASE`, 구글시트 백엔드) → `?mode=getAllData`가 `{logs, goals, tasks}` 반환. `logs`가 이 페이지 데이터.
- **로그 1건 필드**(시트 컬럼, 한글 키): `ID` · `날짜` · `제목` · `내용` · `소요시간` · `태그` · `관련_목표ID`
- **API 모드 4개:** `getAllData`(조회) / `addLog`(생성) / `editItem`(수정) / `deleteItem`(삭제)
- **UI 기능:** ①통계카드 3개 ②기간·키워드 필터 ③날짜별 접이식 타임라인 ④카드별 수정/삭제 버튼 ⑤작성/수정 모달 ⑥`todo_manage`에서 `?related_id=`로 넘어오면 모달 자동 오픈

## C. 데이터 파이프라인 — 추천: worklog.md 직접 읽기 (addLog API 안 씀)
- 관리자 사이트는 GitHub Pages(`admin.cheesehistory.com`)이고 `worklog.md`는 repo 루트에 있음 → 페이지가 **같은 오리진**에서 `fetch('../../worklog.md')`로 바로 읽을 수 있음.
- 이러면 **전송 단계 자체가 불필요.** Claude는 하던 대로 worklog.md에 한 줄 쓰고 push만 하면, 페이지가 파일을 실시간 파싱해 렌더. 저장소가 파일 하나뿐 → **싱크 깨질 여지 0, 진짜 단방향, SSOT 유지.**
- addLog API 대안은 worklog.md + 구글시트 **두 곳 중복 저장** → 드리프트·복잡도만 늘어 기각.
- Jekyll 확인: worklog.md엔 front matter가 없어 Pages가 원문 그대로 서빙 → 읽기 OK. 안전빵으로 루트에 `.nojekyll` 1줄 추가만 권장.

**필요한 포맷 확장:** 현재 규칙 `- YYYY-MM-DD | 요약 | 커밋해시`에 태그 한 칸 추가 →
```
- 2026-07-03 | 요약 한 줄 | #html #js | a1b2c3d
```
파서는 `|` 기준 분리. 이 규칙 변경도 구현 시 포함(Claude가 쓰고 페이지가 읽는 단방향이라 방향성 위배 아님).

## D. 구현 전 결정할 항목 (미결 — 구현 착수 시 확정)
1. **소요시간** — worklog엔 시각 정보 없음. ⓐ 완전 삭제(단순, 추천) / ⓑ worklog 줄에 작업 시작~종료 시각 남겨 자동 추정.
2. **제목 vs 내용 분리** — worklog는 한 줄 요약뿐. ⓐ 요약을 카드 제목으로 쓰고 별도 '내용' 없앰(추천) / ⓑ 유지.
3. **"관련 목표/할 일" 연결 기능** — worklog에 대응 필드 없고 가장 수동적. ⓐ 제거(추천) / ⓑ 유지(그러면 이 필드만 수동 잔존).
4. **통계카드 2·3번**("총 작업 시간"=소요시간 의존, "연결된 할 일"=연결기능 의존). 1·3에서 삭제 시 이 카드도 사라짐 → 대체 지표(예: 총 기록 수 / 최근 작업일 / 이번 주 커밋 수) 정하거나 카드 1개(이번 달 기록)만 남길지.

## E. 구현 중 짚을 오픈 이슈
- **커밋 해시 타이밍:** 해시는 커밋 후 생성. 자기 참조 불가 → "작업 커밋(해시 확보) → worklog 한 줄(그 해시 기입) → worklog만 별도 작은 커밋" 흐름이 가장 깔끔(worklog 셋업 커밋과 동일 구조). 태스크당 커밋 2개가 되는 점 구현 시 확정.
