사용법 — Claude Code에게 말로 지시하면 됩니다

이 화면은 repo의 마크다운(worklog.md · tickets/ · meetings/)을 읽어서 보여주기만 합니다. 여기서 직접 고치지 않고, Claude Code에게 말하면 파일이 자동으로 갱신됩니다. 단 💡 아이디어 탭은 예외 — 시트에 바로 저장되는 캡처 공간이라 여기서 추가·수정·삭제·실현이 됩니다. (아이디어를 프로젝트 티켓으로 올리는 건 회의 때 판단합니다.)

  • 작업 로그 — 작업이 끝나면 자동으로 한 줄 쌓입니다 (날짜·프로젝트·티켓·커밋).
  • 할 일(티켓) 만들기 — "○○ 티켓 만들어줘" → 할 일(todo)로 생성됩니다.
  • 진행 중으로 전환 — "T-003 작업하자" → 진행 중(doing)이 되고, 이후 작업 로그 줄에 T-003이 자동으로 붙습니다.
  • 완료 처리 — "T-003 끝났어" → 완료(done). 진행 중은 한 번에 하나만 둡니다.
  • 회의록 발행 — "오늘 회의 내용 회의록으로 정리해줘" → meetings/에 발행되어 회의록 탭에 표시됩니다.

새 대화를 열면 자동으로 이어집니다. Claude Code로 새 대화(세션)를 시작하면, 지금 진행 중(doing)인 티켓이 무엇인지 자동으로 Claude에게 알려집니다. 그래서 "어디까지 했더라"를 매번 다시 설명할 필요 없이, 이어서 하는 작업이 그 티켓 번호로 계속 기록됩니다. (예: T-002가 진행 중이면 새 대화에서도 T-002로 이어서 태깅) — "끝났어"로 완료하기 전까지 유지됩니다.

프로젝트: 관리자(파랑) / 스토어프론트(주황). 스토어프론트는 private이라 커밋 상세 대신 링크로 열립니다.
