---

치즈랩 회의록을 **기존 시스템 규격 그대로** 발행한다. 아래 절차를 순서대로 지키고, 임의 추론으로 건너뛰지 않는다.

## 1) 입력 확보 — 제일 먼저
이 명령과 함께 받은 내용: `$ARGUMENTS`

- 위가 **회의 원문(붙여넣은 텍스트)**이면 그것을 원문으로 쓴다.
- 위가 **파일 경로**(예: `meetings/2026-07-10.md`, 혹은 임의 위치의 `.md`)면 그 파일을 Read해서 원문으로 쓴다.
- 위가 비어 있으면 **이번 대화 안에서 회의로 정할 내용이 있었는지** 확인한다.
- 그래도 원문이 없으면 **절대 지어내지 말고**, "회의 원문을 붙여넣거나 `.md` 경로를 알려달라"고 되묻고 **중단**한다. (← 예전에 엉뚱한 작업을 하던 원인 차단)

## 2) 규격화 — `meeting-notes-writer` 스킬 사용
반드시 **meeting-notes-writer 스킬**을 불러 그 형식을 적용한다.

- frontmatter 2줄(`date`·`title`) + 고정 6섹션: **배경 / 오늘 정한 것 / 아직 안 정한 것 / 확인된 구조 / 다음 액션 / 이 회의에서 발행한 티켓**.
- 원문 내용을 최대한 살리되, "정한 것 / 아직 안 정한 것"은 원칙대로 정직하게 가른다. 과장·미사여구 금지, 건조하게.
- **발행한 티켓 섹션 빈칸 금지** — 나온 티켓은 번호·제목, 없으면 "없음 (이유)".

## 3) 저장 위치·파일명
- `cheese-blog-parts/meetings/YYYY-MM-DD.md` — 날짜는 **회의 날짜**(원문에 있으면 그것, 없으면 오늘).
- 같은 날짜 파일이 이미 있으면 `YYYY-MM-DD-제목.md` (한글 공백은 `-`).

## 4) 발행 — 커밋 후 푸시까지
1. 회의록 파일을 저장하고 커밋한다. (메시지: `회의록 YYYY-MM-DD 발행 — 제목`) → 짧은 해시 **H** 확보.
2. `worklog.md` 맨 위에 CLAUDE.md 규칙대로 한 줄 추가: `- YYYY-MM-DD | 관리자 | 회의록 YYYY-MM-DD 발행 — 제목 | 티켓 | H`.
   - 티켓 칸: 활성 `doing` 티켓이 있으면 그 번호, 없으면 `-`.
3. `git status`로 올라갈 파일을 사용자에게 보여준 뒤, worklog 줄을 커밋(`worklog: 회의록 YYYY-MM-DD 발행 기록 (H)`)하고 **origin main에 푸시**한다.
4. 푸시 후 안내: 관리자 페이지 **회의록 탭 → 새로고침**(캐시 우회) 누르면 1~2분 내 반영됨.

## 주의
- 회의록은 **결정이 오간 대화**에만. 일상 작업 기록은 worklog 담당(혼동 금지).
- 커밋 메시지는 사용자 승인 없이 hook/서명 우회하지 않는다.
