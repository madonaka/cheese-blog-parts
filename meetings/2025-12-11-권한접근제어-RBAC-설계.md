# 2025-12-11 회의 — 사번·부서·직급별 권한에 따라 화면 접근 제어

> 참여: 치즈 (단독, ChatGPT 상담)
> 주제: 사원·부서·직급·역할 기반으로 관리자 화면 접근을 제어하는 권한 구조(RBAC) 설계
> ※ 이관 노트. Notion 구상 대화(ChatGPT)를 회의록 형식으로 압축. 현 관리자 경영지원 메뉴의 data-roles(EMP·MGR·FIN·ADMIN)가 이 설계의 실현체.

---

## 배경 (왜 이 회의를 했나)

사원 마스터·부서 마스터·사원–부서 매핑·로그인 DB까지 기본 구조는 잡혀 있다. 이 상태에서 "사번·부서·직급별 권한으로 화면 접근을 제어"하려면 세 축만 정리하면 된다: (1) 사람 식별 기준 키 통일, (2) 역할/권한을 어디 두고 어떻게 쓸지, (3) 각 화면이 요구하는 권한 명시.

---

## 오늘 정한 것 (결론)

1. **기준 키 = 이메일이 아니라 employeeId.**
   로그인 계정(login_users)과 사원(employees)을 employeeId FK로 1:1 연결. BUDGET_REQUESTS·EXPENSES에 requesterEmployeeId 컬럼을 추가하고, 이메일은 표시·연락용으로만. 조회도 점차 이메일 대신 로그인 세션의 employeeId 기준으로.

2. **권한 모델 = 확장 가능한 RBAC(Role + Permission).**
   4개 시트로 설계: `login_users`(계정↔employeeId↔기본 Role) · `roles`(EMP/MGR/FIN/ADMIN) · `permissions`(BUDGET_/EXPENSE_/ADMIN_/REPORT_ 세부 행위) · `role_permissions`(롤–권한 매핑). 예외는 옵션 `user_roles`(scope: GLOBAL/DEPT/PROJECT)로.

3. **롤 4종 정의.**
   EMP(본인 예산/지출만 기안·조회) · MGR(부서 예산/지출 승인·조회) · FIN(전체 조회·승인·정산·리포트) · ADMIN(계정·부서·설정 관리 + 전체 조회).

4. **부서·직급은 인사 정보로만 두고 Role은 따로 지정(B안).**
   직급→Role 자동 추론(A안)은 "과장이지만 승인권 없음" 같은 예외 처리가 어려움. 인사 데이터는 사실로 두고 운영 판단(누가 결재권자냐)은 role로 반영 — 정책이 바뀌어도 인사 데이터는 유지.

5. **구현은 심플하게 시작.**
   당장 전부 구현하지 않고 `role` 한 칸 + `if(role === ...)`로 시작. 복잡해지면 `role_permissions`를 추가해 `if(hasPermission('XXX'))`로 승격.

---

## 아직 안 정한 것 (다음에, 필요해지면)

- **RBAC 전면 구현 시점**: 지금은 설계만. permission 테이블로의 승격 타이밍은 화면이 늘어난 뒤.
- **user_roles 예외 확장**: "특정 사원에게만 일시 FIN 권한" 같은 케이스는 나중.
- **초기화 스크립트**: "부장 이상은 기본 MGR" 같은 role 초기 세팅 스크립트는 필요 시.

→ login_users/roles/role_permissions 시트 초안부터 만들고, admin 화면 메뉴를 role/permission 기준으로 숨김/노출.

---

## 확인된 구조 (이번에 OK 난 것)

| 시트 | 역할 |
|------|------|
| login_users | 로그인 계정 ↔ employeeId ↔ primaryRole |
| roles | EMP/MGR/FIN/ADMIN 롤 정의(roleLevel 포함) |
| permissions | 예산/지출 기안·조회·승인, 관리자 화면 접근 등 세부 권한(permGroup·isMenu·pageId) |
| role_permissions | 롤–권한 매핑(allowed) |
| (옵션) user_roles | 예외 권한(scopeType/scopeValue) |

사용 개념: 로그인 → `getCurrentUserContext_(loginId)`로 roles·permissions 배열 확보 → 화면/버튼에서 `permissions.includes('BUDGET_REQUEST_CREATE')`로 노출 제어.
