## 배경
- log_manage 페이지가 방치된 원인은 "전부 수동 입력"이었음.
- 작업 로그의 정본(SSOT)을 repo의 worklog.md로 정하고, Claude Code가 작업 종료 시 자동으로 한 줄 남기게 함.
- 같은 패턴(repo 마크다운 + 관리자 페이지 뷰)으로 티켓·회의록까지 확장하기로 함.

## 오늘 정한 것
- worklog.md = SSOT, log_manage = 단방향 뷰(창문). 양방향 동기화 금지.
- worklog 포맷: `날짜 | 프로젝트 | 내용 | 티켓 | 커밋해시`.
- 프로젝트 구분: 관리자 / 스토어프론트. 한 줄 = 한 커밋 = 한 repo.
- 티켓: tickets/T-NNN.md, frontmatter(id·title·status·project)만. status = todo/doing/done.
- 활성 티켓 = status:doing 티켓(SSOT). 한 번에 doing 하나, 충돌 시 확인.
- 회의록: meetings/YYYY-MM-DD.md, 위 6개 섹션 구조.
- 관리자 뷰: log_manage에 탭 3개(작업 로그/티켓/회의록).
- .nojekyll 추가로 frontmatter 마크다운을 원문 서빙.

## 아직 안 정한 것
- log_manage 원래 개편계획의 소요시간 자동 추정 여부(현재 삭제로 보류).
- 티켓 본문(설명) 필드를 나중에 쓸지.

## 확인된 구조
