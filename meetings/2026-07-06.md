# 2026-07-06 회의 — 세계지도 표시 품질 점검

> 참여: 치즈랩 대표, Claude
> 주제: 근현대 세계지도를 라이브에서 보며 발견한 표시 문제 정리

---

## 배경 (왜 이 회의를 했나)

근현대 세계지도(`?map=world`)를 배포한 뒤 라이브에서 함께 살펴봄.
"배포가 됐는데 바뀐 게 없어 보인다"와 "축소 시 나라 이름이 너무 크다"를 먼저 처리하고,
이어서 대표가 삼국시대 화면과 비교하며 두 가지 표시 품질 문제를 추가로 지적함.

---

## 오늘 정한 것 (결론)

1. **배포는 정상이었음.** 세계지도 자체는 이미 라이브(`@e90458f`)였고, "바뀐 게 없다"고 느낀 이유는
   나라 라벨 수정이 로컬에만 있고 커밋·푸시가 안 돼 프론트에 반영될 게 없었기 때문.

2. **나라 라벨 과대 표시 → 수정·배포 완료.**
   영토 크기에 맞춰 라벨을 줄이는 로직이 `if(ui>1)`(모바일 전용)로 막혀 데스크톱 세계지도에서
   국명이 고정 크기(~19px)로 과대 표시되던 문제. fit-to-country 계산을 모든 화면에 적용.
   큰 영토(삼국 등)는 `16*ku` 상한에 걸려 기존과 동일, 좁은 유럽 소국은 축소/생략.
   - 관리자 뷰어 커밋 `c25dc27` → 스토어프론트 핀 갱신 `a0af28d` → 호스팅 배포 완료(라이브 확인).

3. **남은 두 과제는 T-010으로 발행하고 다음에 진행.** (오늘은 여기까지)

## 아직 안 정한 것 (다음에 = T-010)

### (1) 세계지도 화면 비율 — 삼국시대처럼 크게 보여주기
- 삼국 viewBox 784×516(비율 1.52:1) → 컨테이너 폭 1000px에서 높이 ~658px로 꽉 참.
- 세계 viewBox 2839×1084(비율 2.62:1) → 같은 폭에서 높이 ~382px, 납작한 띠로 보임.
- 세계지도는 본질적으로 가로로 넓어(경도 360° × 위도 134°) 폭을 그대로 두면 높이가 낮음.
- 후보안(택1 또는 조합):
  - (a) **초기 뷰를 세로가 더 큰 크롭으로** 시작(주 대륙 중심) + 축소로 전세계 보기 — 코드만 수정, 에셋 재생성 불필요(가장 가벼움).
  - (b) 지도 컨테이너에 더 큰 높이/최소높이 부여(레터박스 우려).
  - (c) WIN 위도 범위를 좁혀(빈 남빙양·북극 바다 제거) 재빌드 — 지형/해안선/강/영토 path 전부 재생성(무거움).

### (2) 영토와 지형이 어긋남 + 흰 여백이 지저분함
- 관찰: 축소·확대 모두에서 영토 폴리곤이 해안선과 안 맞고, 흰색 빈 영역이 많이 보임(특히 저지대).
- 현 구조상 원인 가설:
  - 해안선 기준은 `assets/land-world.json`(벡터) 하나로 통일하고 지형·강·영토를 여기 클립함.
    그런데 **land-world.json 해상도가 낮아(간략화 과함) 해안이 각지고 블록처럼** 보임.
  - 영토 폴리곤은 **historical-basemaps(다른 출처)** 라서 land 벡터와 육지 범위가 달라,
    영토가 해안까지 못 채우거나 바다로 삐져나가 **가장자리 흰 stroke·빈 슬리버**가 생김.
  - 지형 hypsometric 램프에서 **저지대 = 흰색**(대표 요청 사양)이라, 영토가 안 덮은 평야가 비어 보임.
- 후보 방향:
  - land-world.json을 더 촘촘히 재생성(간략화 완화) → 해안선 매끈하게.
  - 영토 범위를 해안선(land 클립)에 스냅하거나, 영토 저지대에 옅은 fill 보강.
  - 저지대 흰색 톤을 살짝 올려(순백 대신 미색) 빈 느낌 완화.
  - → 대부분 `build/` 에셋 재생성이 필요하므로 별도 작업으로 분리.
