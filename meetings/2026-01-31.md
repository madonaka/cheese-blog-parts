## 배경 (왜 이 회의를 했나)

기존 블로그 이관이 전부 수작업이라 비효율이 컸고, 포스트/이미지 관리 툴도 미완성이었다.
라벨링·UI 배치 같은 단순 반복에 리소스가 새고, 원클릭 발행이 없어 발행 플로우가 직관적이지 않았다.
이미지 워터마크·화질·링크 기준이 없어 일관성이 떨어졌고, 네이버/티스토리 링크가 혼재해 사용자 경험을 해쳤다.
→ 이관 파이프라인(저장→발행→라벨)을 안정화·자동화하고, 핵심 콘텐츠(가계도 시리즈)를 템플릿화해 대량 이관하는 게 목표.

---

## 오늘 정한 것 (결론)

1. **이관을 3단계로 끊는다.**
   1단계 안정화(툴 보완+테스트 이관) → 2단계 가계도(핵심 이관) → 3단계 마감(잔여 처리·품질 점검).
   한 번에 다 옮기지 않고, 시행착오를 앞단계에서 흡수한 뒤 핵심 콘텐츠로 넘어간다.

2. **병행 처리 원칙: 하루 "툴 1개 보완 + 이관 n개".**
   툴 안정성과 이관 진척도를 동시에 확보하기 위한 조합. 툴만 만들다 이관이 멈추거나, 이관만 하다 툴이 방치되는 걸 막는다.

3. **라벨 원클릭 발행 구현 + 작성 UI 박스 재배치.**
   발행 플로우의 가장 큰 병목(원클릭 발행 부재)을 우선 해소한다.

4. **가계도 시리즈 전용 네비게이션·레이아웃을 고정한다.**
   핵심 콘텐츠라 별도 템플릿으로 고정하고, 링크/문구 정책도 통일해 시리즈 일관성을 확보한다.

5. **이미지 정책(워터마크·화질)을 문서화한다.**
   기준이 말로만 있으면 일관성이 안 잡힌다 → 문서로 박제해 산출물에 포함.

## 아직 안 정한 것 (다음에, 필요해지면)

- 이미지 워터마크·화질의 구체 수치 기준 (정책 문서에서 확정)
- 네이버/티스토리 잔여 링크를 어디까지 정리·리다이렉트할지
- 연표 이전 방식 (가계도와 달리 이전 경험이 없음 — PH2에서 재검토)

→ 세부 기준은 PH1 안정화 단계에서 실제 이관을 돌려보며 확정.

## 확인된 구조 (이번에 방향 잡힌 것)

| 항목 | 상태 | 방식 |
|------|------|------|
| 이관 진행 | OK | 안정화 → 가계도 → 마감 3단계 |
| 작업 배분 | OK | 하루 "툴 1개 보완 + 이관 n개" 병행 |
| 발행 플로우 | OK | 라벨 원클릭 발행 + 작성 UI 재배치 |
| 가계도 시리즈 | OK | 전용 네비/레이아웃 고정 + 링크·문구 통일 |
| 이미지 정책 | OK(문서화 예정) | 워터마크·화질 기준 문서로 확정 |

산출물: 포스트 관리 툴(vFinal), 이미지 정책 문서, 이관 체크리스트, 이관 완료된 역사 블로그.
