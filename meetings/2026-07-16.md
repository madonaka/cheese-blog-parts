# 2026-07-16 회의 — 토스페이먼츠 신규 계약 심사 답변 작성·발송

> 참여: 치즈랩 대표, Claude
> 주제: 토스페이먼츠 계약심사 메일(7/14 수신, 미답장)에 스토어프론트 실제 정보로 회신 초안을 만들고 발송한다.

---

## 배경 (왜 이 회의를 했나)

지메일에 토스페이먼츠 관련 메일이 쌓여 있는지 확인해달라는 요청에서 시작. 확인해보니 7/14에 온 "치즈랩(Cheese Lab) 신규 계약 안내" 메일이 미답장 상태였다. 계약심사 필수질문 5개 + 카드사 심사용 홈페이지 준비사항 4개에 답해야 했고, 한 달(대략 8/14) 안에 회신하지 않으면 신청 건이 자동 취소되는 상황이라 시급했다. 대표가 "스토어프론트 실제 정보로 만들어보자"고 해서, 답장 문구를 대충 채우지 않고 실제 라이브 사이트(plus.cheesehistory.com)와 코드를 근거로 채우는 방향으로 진행했다.

---

## 오늘 정한 것 (결론)

1. **답장 항목은 전부 라이브 사이트 실측값으로 채운다.**
   footer.html의 사업자정보는 더미값("김치즈" 등)이었고, 실제 값은 Firestore에서 런타임 로드되고 있었다. 로컬 코드만 보고 답장을 쓰면 틀린 정보(가짜 사업자번호 등)가 나갈 뻔했다. plus.cheesehistory.com을 직접 열어 사업자정보(대표자 김창중·사업자등록번호 840-78-00635 등)·상품 상세 URL·환불정책 문구를 실측했다.

2. **결제모듈은 "직접 연동(Y)"으로 답한다.**
   shop/checkout.html에서 토스페이먼츠 SDK v2를 직접 호출하는 걸 코드로 확인. 카페24·식스샵 등 호스팅사(솔루션사)를 안 거치므로, 질문 5번은 Y로 명확히 답할 수 있었다.

3. **단건 최고가는 지금 그대로 6,000원으로 신고한다.**
   현재 라이브 상품은 "한국사-미국사 비교연표" 1개(6,000원)뿐. 향후 확장을 감안해 여유 있게(3만원 등) 신고하는 안도 검토했으나, 대표가 "현재가 그대로"를 선택. 나중에 더 비싼 상품이 나오면 재신고가 필요할 수 있다는 걸 인지한 채로 결정.

4. **환불정책 URL은 이용약관(제6조)과 상품 상세 "환불 정책" 탭 둘 다 넣는다.**
   PPT는 상품 상세 탭만 캡처했지만, 이메일 답변은 굳이 PPT 범위에 맞출 필요 없다고 판단. 법적 근거(약관)와 소비자가 실제 마주치는 화면(상품 상세)을 함께 보여주는 게 더 완전한 공시라는 게 이유. 두 문서의 환불 조건 문구가 표현은 다르지만(법률 조항 vs 소비자 친화적 요약) 실질적으로 모순은 아니라고 판단해 그대로 유지.

5. **결제경로 PPT는 재작업하지 않고 기존 산출물을 그대로 쓴다.**
   `Cheese_History_Plus_홈페이지_결제경로.pptx`(Drive, 7/9 제작)가 이미 T-017에서 완료된 결과물이었다. 회원/비회원 구매 흐름·하단 정보·환불규정 캡처까지 39페이지로 다 되어 있어 새로 만들 필요가 없었다.

6. **메일 발송은 대표가 직접 한다.**
   Claude가 Gmail 임시보관함에 답장 초안(원본 스레드 회신으로 연결)까지만 만들고, 첨부파일(PPT)·최종 검토·발송은 대표 몫으로 남겼다. 초안 작성 도구가 첨부파일을 지원하지 않아 PPT는 대표가 직접 Drive에서 붙였다. 최종 검토 중 이용약관 URL 표기가 꺾쇠괄호로 중복 표시되는 서식 문제를 발견해 대표가 직접 수정했고, 이후 발송을 완료했다.

## 아직 안 정한 것 (다음에, 필요해지면)

- 이용약관 제6조와 상품 상세 페이지 환불 정책 탭의 문구를 하나로 통일할지 — 지금은 모순은 아니라고 보고 그대로 뒀지만, 법적 리스크를 더 줄이려면 다듬을 여지가 있음.
- 상품 상세 페이지의 미리보기(시용상품) 이미지를 "구매 전 미리보기"로 명시적으로 라벨링할지 — 청약철회 제한을 더 안전하게 주장하기 위한 보완책으로 언급만 됨.
- 6,000원보다 비싼 상품을 낼 계획이 확정되면 최고가 재신고 필요.

→ 토스페이먼츠 측 카드사 심사 결과(추가 요청·반려 여부)를 보고, 필요해지면 그때 다시 다룬다.
