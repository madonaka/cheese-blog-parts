# 2025-12-11 회의 — 관리자페이지 내 배치 프로그램 활용법 구상

> 참여: 치즈 (단독, ChatGPT 상담)
> 주제: 관리자 시스템에 배치(스케줄) 작업이 필요한지, 무엇을 배치로 빼고 무엇으로 돌릴지
> ※ 이관 노트. Notion 구상 대화(ChatGPT)를 회의록 형식으로 압축. 히타치 현장의 배치 문화를 내 시스템에 작게 이식해보려는 동기.

---

## 배경 (왜 이 회의를 했나)

히타치 현장에서 배치로 돌리는 작업이 많은 걸 보고, 지금 구상 중인 관리자 페이지에도 배치를 적용해볼 수 있을지 검토했다. "화면 열고 바로 끝나는 작업(온라인 처리)"과 "시간 기준·반복·여러 건 일괄(배치)"을 나누는 게 핵심.

---

## 오늘 정한 것 (결론)

1. **온라인 처리로 둘 것 vs 배치로 뺄 것을 구분.**
   온라인(그대로): 예산/지출 등록·수정, 직원 정보, 개별 PDF 생성, 단일 승인/반려. 배치 후보: 급여·정산 마감, PDF·다운로드 링크 발송(+만료), 결재 미승인 알림·주간 요약, 로그·백업/아카이브, 통계/대시보드 집계, 외부 서비스 동기화, 데이터 정합성 체크.

2. **배치 실행 수단 = Apps Script "시간 기반 트리거"가 1순위.**
   현 스택(구글 시트 + Apps Script)에 가장 자연스러움. 일반 함수(nightlyJob 등)를 만들고 트리거로 매일/매시간/매주 스케줄. 역할 분리: 화면 호출 = doGet/doPost, 자동 배치 = 시간 트리거 + 일반 함수.

3. **배치는 업무 단위로 쪼개 명명.**
   dailyBackupJob · payrollClosingJob · orderMailSenderJob · approvalReminderJob 같은 식.

4. **관리자 페이지엔 배치 실행 이력 + 재실행 버튼.**
   각 배치의 실행 이력(성공/실패·건수) 표시, "지금 바로 재실행"·"실패분만 재실행" UI.

5. **첫 배치 도입 순서 추천.**
   ① 결재 미승인 알림(가볍고 효과 큼) → ② 주간 요약 리포트 → ③ 네이버 스토어 주문→메일 발송(수익 연결·연습 좋음) → ④ 급여 마감 / 백업 / 정합성 체크 순.

---

## 아직 안 정한 것 (다음에, 필요해지면)

- **GCP(Cloud Scheduler + Cloud Functions) / 외부 서버 cron**: 확장성은 좋으나 지금은 오버스펙 — 사용자·처리량·외부 연동이 커지면 그때 고려(추측).
- **실제 첫 배치 선정·구현 착수**: 추천 순서는 있으나 착수 미정.

→ "매일 03시 미승인 결재 메일" 같은 배치를 Apps Script 트리거 + 의사코드로 설계하는 게 다음 단계.

---

## 확인된 구조 (이번에 OK 난 것)

| 구분 | 처리 방식 | 예 |
|------|-----------|-----|
| 온라인(OLTP) | 화면 버튼 → Apps Script 즉시 | 등록·수정·개별 PDF·단일 승인 |
| 배치 | 시간 트리거 + 일반 함수 | 급여 마감·주문 발송·링크 만료·알림·백업·집계·정합성 |

관리 화면: 배치 실행 이력(성공/실패·건수) + 재실행/실패분 재실행 버튼.
