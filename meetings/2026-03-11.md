## 배경 (왜 이 회의를 했나)

블로그 자료 판매를 제대로 사업화하려는데, 기존 외부 플랫폼이 다 걸림돌이었다.
'크몽'은 수수료가 과하고 할인 기능이 없으며, '네이버 스토어'는 디지털 파일 다운로드를 지원하지 않아 파일을 수동으로 전달해야 했다.
즉 어느 쪽이든 수수료·심사·수동 관리로 비효율이 컸다.
→ 파일 다운로드가 되고 할인 같은 마케팅 기능을 갖춘, 로그인·결제·다운로드가 간편하게 자동 처리되는 **자체 스토어**를 만들자는 것.
역사블로그의 트래픽을 구매로 연결하는 게 핵심이며, 원칙은 "최소 법적 리스크 + 최대 효율(자동화)".

---

## 오늘 정한 것 (결론)

1. **별도 '치즈의 스토어'를 자체 개발한다.**
   외부 플랫폼에 얹지 않고, 구글 안티그래비티(Antigravity)로 독립 스토어를 구축한다.

2. **관리자페이지 ↔ 치즈의 스토어 이원화 전략.**
   관리(백오피스)와 판매(고객 대면)를 한 덩어리로 만들지 않고 둘로 분리해 각자 최적화한다.

3. **이원화 아키텍처를 계층별로 확정한다.**
   - 프론트엔드: GitHub / Cloudflare
   - 백엔드: Google Apps Script / Firebase
   - DB: Google Sheets / Firestore

4. **목표 = 로그인·결제·다운로드 전 과정의 간편·자동 처리.**
   구매까지 필요한 모든 과정이 손 안 대도 흘러가게 만든다.

5. **블로그 트래픽을 구매로 전환한다.**
   이미 있는 역사블로그 방문자를 스토어 구매로 이어붙여 사업화한다.
