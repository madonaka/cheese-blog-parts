# 2025-12-01 회의 — 치즈 역사블로그 테마 부품화 PJ (계획)

> ※ 이관 노트. Notion "[251201][계획] 테마 부품화 PJ"를 회의록 형식으로 옮긴 것. 이 레포(cheese-blog-parts)의 설계 원본. 진척 체크는 2025-12-01 기준.

---

## 배경 (왜 이 회의를 했나)

현재 테마는 사실상 "치즈 블로그 플랫폼"이 하나로 떡 붙어 있지만, 구조 자체는 이미 `cheese-*` 접두어로 컴포넌트화돼 있다. 이걸 기능 단위로 쪼개 GitHub 레포에 올리고, 테마에서는 그때그때 필요한 부품만 jsDelivr로 끌어다 쓰는 구조로 만드는 것이 목표. 테마를 통째로 바꿔도 이 부품 링크만 유지하면 치즈 컴포넌트는 그대로 살아남는다.

---

## 오늘 정한 것 (결론)

1. **부품화 기준 = 기능 단위.**
   JS·CSS·HTML이 강하게 엮인 하나의 "기능"을 1부품으로 본다(퀴즈·라이트박스·푸트노트·인포박스·배지·시대연표 등). `cheese-*` 접두어가 붙은 건 다른 테마에서도 재사용 가능한 범위라 통째로 외부화.

2. **부품 내부는 3레이어 세트.**
   각 부품은 `CSS + JS(전역 이름 하나만 노출, 예: window.CheeseQuiz) + README(HTML 예시)` 3종 세트로 구성.

3. **레포 구조 = cheese-blog-parts 폴더별 모듈.**
   `core/`(선택) · `quiz/` · `quiz-image/`(서브 모듈) · `lightbox/` · `footnote/` · `components/`(infobox·badge·notice-button·era-timeline, CSS-only) · `examples/`. jsDelivr 한 레포에서 전부 뽑아 쓰고 버전 태깅도 한 번에.

4. **버전 전략 = 태그 버전으로만 링크.**
   jsDelivr는 항상 `@v1.0.x` 태그로 링크(main 직결 금지). 메이저 1.x는 HTML 구조를 깨지 않게 유지, DOM 구조·데이터 속성 변경이 필요하면 v2.0.0으로 새 태그.

5. **마이그레이션 순서 = 독립적인 것부터.**
   Step0 테마 XML 백업 → Step1 Lightbox/Infobox/Badge/Era Timeline → Step2 Footnote → Step3 Quiz Core → Step4 Quiz Image.

6. **테마에 최종적으로 남길 것.**
   VTrick 레이아웃/위젯 코드 + 부품용 HTML 뼈대(모달·라이트박스 컨테이너) + jsDelivr link/script 태그 + 테마 특화 스크립트(스크롤탑·목차 토글 등)만. 치즈 부품의 스타일/로직은 전부 레포에서 관리.

---

## 아직 안 정한 것 (다음에, 필요해지면)

- **Core 유틸 분리**: 지금 core가 많지 않아 초반엔 생략, 나중에 정리.
- **공통 부분 처리**: 기능 간 겹치는 부분을 `core`로 뺄지 / 각 기능에 복사해 넣을지는 마이그레이션하며 판단.
- **JS/data 속성 충돌 방지**: 부품 분리 후 class·data- 속성이 겹치지 않게 유지.

→ 어느 부품부터 실제로 뽑을지 정한 뒤, 원래 코드를 "외부 모듈용 코드"로 다시 정리하며 진행.
