# 2025-12-08 회의 — 자동 메일 발송 트리거 기능 구상

> 참여: 치즈 (단독, ChatGPT 상담)
> 주제: 상품 판매 시 지정 PDF 자동 발송, 파일 교체, 다운로드 기간 제한이 가능한지 구상
> ※ 이관 노트. Notion 구상 대화(ChatGPT)를 회의록 형식으로 압축. 원문에 Apps Script 예시 코드 다수 포함(여기선 접근법만 요약).

---

## 배경 (왜 이 회의를 했나)

관리자페이지에서 "어떤 상품이 팔렸을 때 어떤 PDF를 보낼지 지정하고, 필요에 따라 파일을 교체"할 수 있는지 구상. 이어서 "다운로드를 계속 열어두면 무제한 복제될 것 같은데, 일정 기간만 다운로드하게 할 수 있나?"로 확장됐다.

---

## 오늘 정한 것 (결론)

1. **정석 구조 = 상품↔PDF 매핑 테이블 + 관리자 편집 화면.**
   `products` 시트(productId·productName·fileId·active·updatedAt·note)를 매핑 DB로 두고, 주문 처리 로직이 상품키(상품명/코드)로 fileId를 조회해 발송. **파일 교체 = 시트 fileId만 바꾸면 이후 주문부터 새 파일**이 나감.

2. **관리자 페이지에서 매핑 조회/수정.**
   Apps Script `doGet(mode=productList)`로 목록 조회, `doPost(mode=updateProductFile)`로 fileId 갱신(드라이브 URL 넘기면 파일ID만 추출). 시트 직접 편집 대신 UI로 관리.

3. **다운로드 기간 제한은 가능(복제 자체 차단은 불가).**
   PDF는 드라이브에만 두고 링크만 발송. `orders`에 sentAt·expireAt 기록 → 타임 트리거가 만료된 주문의 드라이브 공유를 "링크 접근 → 비공개(Restricted)"로 전환. 단, 이미 받은 파일을 되돌릴 순 없음("복제 완전 차단"은 불가능, "퍼나르기 어렵게 + 새 다운로드 차단"이 현실적 목표).

4. **공평한 만료 = 주문별 expireAt + 잦은 트리거.**
   expireAt을 "구매시점 + n시간"으로 주문별 기록하고, 트리거를 10~30분 간격으로 돌리면 사람마다 오차 ±10~30분 수준. 상품/이벤트별로 만료시간 차등(예: Basic 24h, 특가 1h)도 가능.

---

## 아직 안 정한 것 (다음에, 필요해지면)

- **만료 기간 정책**: 7일 / 24시간 / 상품별 중 무엇으로 할지 미정.
- **트리거 주기**: 10·30·60분 중 미정(오차 vs 부하 트레이드오프).
- **강화 옵션 적용 여부**: 주문별 개인 파일 복제(주문번호·이메일 섞기)+만료 후 삭제, 구매자 워터마크 — 재직증명서 "템플릿 복사→개인 파일" 로직 재활용 가능하나 도입 미정.
- **과거 주문 추적**: `orders`에 sentFileId 컬럼 둘지.
- **Workspace expirationTime 기능**: 개인 Gmail 가용 여부 불확실(추측), 로그인 기반이라 UX 복잡 → 링크 비활성화 방식이 더 적합할 가능성.

→ "구매 후 24시간 + 30분 트리거" 같은 조건으로 실제 코드 한 세트 설계는 다음 단계.

## 확인된 구조 (이번에 OK 난 것)

| 구성 | 역할 |
|------|------|
| `products` 시트 | 상품–파일 매핑 DB (productId·productName·fileId·active·updatedAt·note) |
| 주문 처리 로직 | 상품키로 fileId 조회(getFileIdByProduct_) → 발송(sendPdfToBuyer_) |
| 관리자 페이지 | 매핑 목록 조회/수정 UI (Apps Script API) |
| 만료 처리 | orders(sentAt·expireAt) + 타임 트리거 → setFilePrivate_(드라이브 공유 차단) |

가능/불가능: 기간 제한·주문별 파일·워터마크는 가능 / 이미 다운로드된 파일의 사후 무효화는 불가능.
