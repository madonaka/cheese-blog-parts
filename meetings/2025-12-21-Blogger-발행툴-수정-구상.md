# 2025-12-21 회의 — Blogger 발행 툴 수정 구상

> 참여: 치즈 (단독, ChatGPT 상담)

> ※ 이관 노트. Notion 구상 대화(ChatGPT)를 회의록 형식으로 압축.

## 배경 (왜 이 회의를 했나)

현재 POSTS 시트 → 미리보기 → Blogger 발행/업데이트까지는 된다. 이제 이걸 "관리자 CMS"로 업그레이드해 콘텐츠 제작과 발행을 분리하고, 상태·검색·라벨·SEO를 갖춘 운영형 도구로 만드는 단계. 태그(라벨) 지정 가능 여부도 확인했다.

---

## 오늘 정한 것 (결론)

1. **수정 흐름을 2단계로 분리.**
   ① DB 수정: 관리자에서 title/intro/body 편집 → 저장(POSTS 갱신, updated_at 자동). ② 블로그 반영: 저장된 내용을 Blogger에 업데이트/발행. "콘텐츠 제작 ↔ 발행"을 분리해 실수를 줄인다.

2. **발행 상태 관리 컬럼 통일.**
   blogger_post_id · blogger_url · status(draft/scheduled/published/archived) · publish_at · last_published_at · last_synced_at · sync_state(ok/error/pending) · sync_error. "왜 발행 안 됐지?"가 바로 보이게.

3. **발행 버튼 분리 + 중복 발행 방지.**
   초안 저장(update만) / 공개 발행(update+publish) / 초안 되돌리기(revert — Blogger API revert 엔드포인트). 같은 post_id에 blogger_post_id가 있으면 insert 금지, update만 강제(중복 글 생성 차단).

4. **목록/검색/필터.**
   POSTS 목록(제목·ID·상태·수정일·발행일), 제목/본문 검색, 상태 필터(draft/published), 최근 수정·발행순 정렬, "미발행만" 체크.

5. **태그(라벨) 지정 가능.**
   Blogger는 Labels로 관리, API payload에 `labels` 배열 전달. POSTS에 `labels` 컬럼(쉼표 저장) → split/trim해서 배열로.

6. **SEO/템플릿 자동화 + 안전장치.**
   slug·meta_description·keywords·og_image 컬럼, h2 기반 TOC 자동 생성, 템플릿 블록·대표이미지 자동 삽입. 안전: 발행/삭제는 ADMIN 롤, 공개 발행 2단 확인, body의 `<script>` 등 위험 태그 차단.

## 아직 안 정한 것 (다음에, 필요해지면)

- **태그 입력 방식**: 쉼표 입력 vs 칩(chip) UI.
- **편집기**: HTML 직접 입력 유지 vs 간단 WYSIWYG(굵게/링크/리스트).
- **선택 확장**: 예약 발행(publish_at 트리거), 에러 재시도 버튼, 내부 링크 자동 추천(같은 라벨 3개), SEO 체크리스트, 변경 이력(body_prev·updated_by·reason).
- **먼저 붙일 것**: 목록/검색 vs 수정/저장 vs 초안 되돌리기 중 택1.

→ 우선순위대로 목록→편집→발행 순으로 붙인다.

## 확인된 구조 (이번에 OK 난 것)

- API mode 설계: getPost · updatePost · listPosts · publish · revertToDraft.
