# 2026-07-13 회의 — 인터랙티브 아트 실험(lumen)과 소련편 시리즈 티저 제작

> 참여: 치즈랩 대표, Claude
> 주제: 프롬프트로 만드는 "스크롤을 멈추게 하는 인터랙티브 웹사이트"를 실험하고, 쓸 만하면 스토어프론트에 적용

---

## 배경 (왜 이 회의를 했나)

대표가 영상에서 본 인터랙티브 웹사이트 결과물에 감탄해, 같은 프롬프트("세계 최고 수준의 인터랙티브 아트 — 스크롤을 멈추게 하는 시청각 경험")로 직접 시도해보고 싶다고 함.
먼저 blog-parts 쪽에 실험 부품을 하나 만들어보고, 영상 속 결과물과 비슷하면 스토어프론트에 적용하는 걸 전제로 시작.

---

## 오늘 정한 것 (결론)

1. **실험 부품(lumen)을 blog-parts에 먼저 만들어 검증한다.**
   `cheese-blog-parts/lumen/` = "빛의 역사" 스크롤 시네마. 순수 Canvas 2D + WebAudio, 라이브러리 0, 단일 HTML.
   스토어프론트 이식이 쉬운 구조로 잡음(정적 파일 하나). 결과가 기대 이상이라 다음 단계로 진행 결정.

2. **적용처는 "파는 페이지"가 아니라 "갖고 싶게 만드는 페이지".**
   shop 구매 동선(products·detail·checkout)과 학습 뷰어 본체에는 넣지 않기로 함 — 몰입 연출이 구매 결정을 늦추거나 콘텐츠 소비를 방해하기 때문.
   대신 **시리즈 티저**가 최적. 역사 콘텐츠는 "시대의 공기"를 파는 것이라 이 연출이 전환에 직결됨.

3. **첫 적용 대상 = 소련편 티저.**
   콘텐츠가 가장 많은 시리즈라 프로토타입 대상으로 선택. 잘 나오면 나머지 시리즈(미국·중화민국·가계도)에 템플릿으로 복제.
   `cheese-storefront/series/soviet/` 에 「어느 이상의 일생」제작 — 17편 서사를 12개 장면의 스크롤 아크로 옮김(잉크 → 눈 위의 피 → 붉은 쐐기 → 회색 얼룩 → 철의 장막 → 동상 전도 → 쿠바 13일 → 정체 → 균열 → 깃발 하강 → 17편 목록 + CTA).
   시리즈 설계도의 척추("이상은 뜨거웠고 결과는 차가웠다")를 시각 언어로 따름.

4. **스크롤 페이싱은 "빠르지도 느리지도 않게" 반복 튜닝.**
   초기엔 스크롤이 너무 무거웠고(1900vh), 조정 후엔 너무 빨랐음. 최종적으로 소련편은 장면 수(12개)와 서사 밀도가 높아 lumen(9장면)보다 더 머무는 페이싱으로 확정.
   조정 손잡이 3개: 전체 길이(SPACE_VH) · 휠 증폭배수 · 스크롤 스무딩. 값은 감상하며 계속 미세조정.

---

## 아직 안 정한 것 (다음에, 필요해지면)

- 소련편 티저를 실제로 스토어프론트에 게시(배포)할지 — 지금은 미커밋 프로토타입, 대표 최종 감상 판정 대기
- 나머지 시리즈(미국·중화민국·가계도) 티저로 확장할지, 소련편 하나로 반응부터 볼지
- index 홈 히어로 첫 화면 / learn 완주 에필로그(outro)에도 이 연출을 이식할지 (Claude 제안만 된 상태)
- 페이싱 최종값 (SPACE_VH 등) — 대표가 실기기에서 굴려보고 확정

→ 소련편 티저에 대한 대표 판정이 나오면 그걸 기준으로 확장 여부·게시 여부 결정.

| 항목 | 상태 | 방식 |
|------|------|------|
| 실험 부품 lumen | OK | blog-parts/lumen, 단일 HTML(Canvas+WebAudio), 라이브러리 0 |
| 소련편 티저 | OK | storefront/series/soviet, 12장면 스크롤 아크 + 17편 목록·CTA |
| 검증 방식 | OK | `?headless` 심(rAF→setTimeout) + 픽셀 샘플링으로 숨김 렌더러에서도 검증 |
| 성능·접근성 | OK | glow 스프라이트 캐시, safe-area, 휠 증폭, prefers-reduced-motion 등 반영 |
| 이식성 | OK | 정적 파일 하나라 storefront에 폴더째 얹으면 끝 |
