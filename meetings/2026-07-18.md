# 2026-07-18 회의 — 시리즈 복습 자료실(학습–요약–문제 루프)

> 참여: 치즈랩 대표, Claude
> 주제: `/korean/series.html`에 등록된 문제·퀴즈를 바로 풀 수 있게 잇는 탭/자료실을 만들어, 인강 사이트처럼 "돈 내고 볼 만한 구조적 연결"을 갖춘다

---

## 배경 (왜 이 회의를 했나)

대표가 시리즈 페이지에서 각 회차의 문제·퀴즈·요약을 바로 풀거나 볼 수 있도록 모아두는 탭/자료실을 원함. 목표는 단순 기능 추가가 아니라, "콘텐츠 학습 → 요약 → 추가공부"가 한 화면에서 순환하는 인강형 구조를 만들어 사람이 돈을 내고 볼 이유를 만드는 것. 이번 초점(HP 고도화 · 카드학습 기능 고도화)과 재방문·전환에 직접 닿는 사안이라 회의로 남김.

현재 구조 확인 결과: 시리즈 페이지 카드에서 나가는 링크는 `/learn/viewer.html?id=` 하나뿐이고, 퀴즈로 가는 동선은 학습 뷰어 아웃트로를 거쳐야만 있음. 시리즈→퀴즈 연결 필드도 `learn_sequences.cta_quiz_set_id`(역방향 하나)뿐. 요약(`full_text`)은 뷰어 모달 안에만 갇혀 있음.

---

## 오늘 정한 것 (결론)

1. **자료실을 독립 페이지로 빼지 않고, series.html 안에 2층 구조로 넣는다.**
   돈 내는 이유는 콘텐츠 자체가 아니라 "여기서 완결되는 학습 루프"이므로, 강의–요약–문제–오답이 한 화면에서 보이게 시리즈 페이지 안에 얹는다. 별도 자료실 페이지는 루프를 오히려 끊음.

2. **1층 = 콘텐츠 목록 탭의 에피소드 카드에 복습 칩 추가.**
   지금 카드의 링크가 "학습하기" 하나뿐인데, 여기에 "요약노트"·"퀴즈" 칩을 나란히 붙여 회차 하나가 `강의 | 요약 | 문제` 세트로 보이게 한다. 학습을 마친 회차는 퀴즈 상태/점수가 칩에 뜨고, 안 푼 회차는 "퀴즈 풀기"로 남겨 다음 행동을 유도.

3. **2층 = 새 "복습 자료실" 탭.** 콘텐츠 목록 옆에 두고 3섹션 구성:
   - 나의 복습 현황(학습 완료 편수 / 퀴즈 완료 편수 / 오답노트 바로가기)
   - 회차별 복습(회차마다 요약노트·퀴즈를 한 줄로)
   - 시리즈 종합 문제(회차 무소속 전 범위 모의고사·플래시카드) — 완주자용, 프리미엄 가치가 가장 높은 자리.

4. **단계적으로 간다 — 1단계는 스키마 변경 없이 지금 데이터로 즉시 가능.**
   series.js가 이미 시리즈의 `learn_sequences` 문서를 통째로 로드하므로 `full_text`(요약)·`quizzes[]`·`cta_quiz_set_id`가 다 메모리에 있음. 요약은 모달/페이지로 띄우고 퀴즈는 `/quiz/quiz-detail.html?id=` 링크만 만들면 됨. 퀴즈 카드 UI도 `quiz-hub-panel.js`(허브용 퀴즈 탭 모듈) 재사용 가능.

5. **차별화의 핵심은 "오답 → 강의 회귀 루프"다.**
   이미 `remedial_set_id`·`remedial_mode`(보충 학습 연결) 자산이 뷰어 안에 숨어 있음. 오답노트에서 "이 문제는 N편 ○○ 부분 → 해당 지점으로 이동" 딥링크가 되면, 문제은행 사이트(강의 없음)·일반 인강(오답→강의 회귀 없음)이 못 하는 지점이라 이게 셀링 포인트가 된다.

6. **게이팅은 기존 1회 구매 모델 그대로, 맛보기는 후하게.**
   무료 회차(`free_episode_count`)의 요약·퀴즈는 전부 공개, 잠긴 회차는 문항 수·난이도까지만 보여주고 자물쇠 처리. "자료가 이만큼 쌓여 있는데 잠겨 있다"가 보이는 것이 전환의 핵심. 구독 개념은 도입하지 않음.

---

## 아직 안 정한 것 (다음에, 필요해지면)

- 탭 이름: "복습 자료실" vs "문제·요약" — 은유 없이 행동이 보이는 이름으로. 1단계 구현하며 확정.
- 요약노트를 독립 페이지(`/learn/summary.html?id=`)로 승격할지 — 상품 구성물("요약노트 8부")로 셀 수 있게 되지만, 잠긴 회차의 `full_text`가 이미 클라이언트로 내려오는 현재 구조상 요약 노출 정책(UI 게이팅으로 충분한지)을 그때 짚어야 함.
- `quiz_sets`에 `series_id` 필드를 추가할지(종합 문제 섹션용) — 지금은 시리즈→퀴즈 연결이 역방향 하나뿐이라 회차 무소속 문제를 걸 수 없음. 관리자 편집기에 시리즈 드롭다운 추가 필요.
- 퀴즈 점수 표시·오답→강의 딥링크의 결과 저장 스키마 — 공수가 가장 크고 별도 설계 필요.

→ 1단계(자료실 탭 + 복습 칩)를 먼저 붙여 반응을 보고, 요약 페이지 승격 / series_id / 점수·딥링크 순으로 확장 여부를 결정.
